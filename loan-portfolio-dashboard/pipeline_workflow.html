Step 0
Portfolio Interview
— no script — Claude conducts interview —
📋 User-confirmed distribution settings
Ask the user about product mix (Personal/Home/Auto/Business), employment type split (Salaried/SEP/Business Owner), city tier distribution (Tier 1/2/3), loan amount ranges per product, and total record count.

Outputs: CLI arg string passed to Step 1
🛑
Mandatory Stop
Wait for user to confirm portfolio distribution before generating data
Step 1
Generate Loan Applications
generate_loan_applications.py
📄 loan_applications.xlsx
Generates 3,000 synthetic Indian loan application records using parameterised distributions from the Step 0 interview. Includes App_ID, Age, Gender, City, City_Tier, Employment_Type, Monthly_Income, CIBIL_Score, Loan_Product, FOIR, and more.

CLI args: --records, --product-weights, --emp-weights, --tier-weights, --loan-amt-*
Step 2a
Build Loan Pricing Table
build_loan_pricing.py
📄 Loan Pricing.xlsx
Generates 810 rate combinations across 5 risk dimensions: CIBIL band, Employment Type/Category, Loan Amount range, Tenure range, and Age group. Base Rate 9.00%. Final rates 9.25%–15.00%, colour-coded green/yellow/red.

Sheets: Pricing Table (810 rows), Rate Summary (pivot)
🛑
Mandatory Stop — Pricing Review
User opens Loan Pricing.xlsx, reviews/adjusts Final Rate (%) column, then saves and closes
Step 2b
Build Eligibility Rules
build_loan_eligibility_rules.py
📄 Loan Eligibility Rules.xlsx
Generates 8 BRE eligibility rules (R001–R008) with editable amber Value cells.

R001 Loan Product · R002 CIBIL Min · R003 FOIR Salaried · R004 FOIR Non-Salaried · R005 Max Loan Amount · R006 City Tier · R007 Min Age · R008 Max Age

Sheets: Eligibility Rules (editable), Rule Summary (reference)
🛑
Mandatory Stop — Rules Review
User reviews/adjusts amber Value column for R001–R008, sets Enabled=No for any rule to skip
Step 3
Generate Approved Loans
generate_approved_loans.py
📄 approved_loans.xlsx
Reads loan_applications.xlsx, applies BRE rules from Loan Eligibility Rules.xlsx, and looks up interest rates from Loan Pricing.xlsx. Adds Loan_ID, Approval_Date, Sanctioned_Amount, Interest_Rate, EMI, Processing_Fee.

Typical approval rate: ~5% of applications
Sheets: Approved_Loans, Summary
Step 3b
Build Repayment Assumptions
build_repayment_assumptions.py
📄 Repayment Assumptions.xlsx
Generates a 6-section editable config file with all repayment simulation assumptions.

§1 Payment Profile Distribution (Pristine/OccDelay/Chronic/PartialPrepay/FullPrepay)
§2 DPD Distribution — Occasional Delay
§3 DPD Distribution — Chronic Delay
§4 Partial Prepayment parameters
§5 Full Prepayment parameters
§6 Write-offs & Penal rules (NPA DPD, penal rate, default penalty)
🛑
Mandatory Stop — Repayment Config Review
User reviews all 6 sections (amber cells), ensures % columns in §1–§3 sum to 100
Step 4
Generate Repayment Schedule
generate_repayment_schedule.py
📄 loan_repayment_schedule.xlsx
Reads approved_loans.xlsx and simulates monthly repayment for all loans. Applies 5 payment profiles, calculates DPD/DPD Bucket, penal interest, prepayment charges, default penalties.

21 output columns: Schedule_ID, Loan_ID, EMI_Number, Due_Date, DPD, Opening_Balance, Scheduled_EMI, Penal_Interest, Prepayment_Amount, Loan_Status …
DPD buckets: Current · 1-30 · 31-60 · 61-90 · 90+
Step 5
Repayment Dashboard
generate_repayment_dashboard.py
📊 repayment_dashboard.xlsx
Reads loan_repayment_schedule.xlsx and builds a formula-driven analytics dashboard.

§1 Executive KPIs (total EMIs, collected, penal interest, prepayments, NPA count)
§2 Payment Status Distribution
§3 DPD Bucket Analysis
§4 Loan Status Summary
§5 Prepayment Analysis
Step 6
Lifecycle Dashboard
generate_lifecycle_dashboard.py
📊 loan_lifecycle_dashboard.xlsx
Reads all three pipeline files and builds a unified cross-file lifecycle view.

§1 Lifecycle KPIs (approval rate, collection efficiency, portfolio yield)
§2 Applications by Lead Source
§3 Approved Loans by Employment Type
§4 Approved Loans by City Tier
§5 CIBIL Band Analysis
§6 Repayment Performance by Payment Status

Sheets: Dashboard + 3 data sheets (Applications, Approved_Loans, Repayment_Schedule)
🛑
Mandatory Stop — Target KPI Interview
Ask user for target values for all 10 Executive KPIs before generating final dashboard

generate_dashboard.py

Final output — consolidates the full pipeline into one workbook.

📋 Data — 1,000 synthetic portfolio records
